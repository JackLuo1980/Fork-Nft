# Fork-Nft 重构开发计划

> 创建时间: 2026-03-21
> 目标: 功能与 flux-panel 完全一致，增加 realm 和 nft 协议支持，优化 UI 交互和监控信息

## 一、项目现状分析

### 1.1 技术架构

| 组件 | 技术栈 | 状态 |
|------|--------|------|
| 前端 | React 18 + Vite + TypeScript + shadcn-bridge | ✅ 已实现 |
| 后端 | Go 1.24 + GORM + net/http | ✅ 已实现 |
| 数据库 | SQLite / PostgreSQL | ✅ 已实现 |
| 转发引擎 | go-gost + nftables + realm | ✅ 部分实现 |
| 部署 | Docker + Docker Compose | ✅ 已实现 |

### 1.2 数据模型分析

**核心表**:
- `user` - 用户表
- `forward` - 转发表
- `tunnel` - 隧道表
- `node` - 节点表
- `user_tunnel` - 用户隧道关联表
- `forward_port` - 转发端口表
- `speed_limit` - 限速表
- `user_quota` - 用户配额表
- `chain_tunnel` - 链式隧道表
- `tunnel_group` - 隧道组表
- `user_group` - 用户组表
- `group_permission` - 分组权限表

### 1.3 已实现功能

#### ✅ 基础功能
- 用户管理（CRUD）
- 节点管理（CRUD）
- 隧道管理（CRUD）
- 转发管理（CRUD）
- 分组管理（用户组/隧道组）
- 分组权限管理
- 流量统计
- JWT 认证
- WebSocket 实时通信

#### ✅ 引擎支持
- gost 引擎（完整实现）
- nftables 引擎（基础实现）
- realm 引擎（基础实现）
- 引擎选择功能（engine 字段）

#### ✅ 其他功能
- 面板共享（联邦）
- Peer Share
- 监控权限
- 公告系统
- 批量操作
- 诊断功能

### 1.4 待完善功能（与 flux-panel 对比）

#### ❌ 需要补充的功能
1. **端口转发与隧道转发模式切换**
   - 当前缺少明确的转发模式区分
   - 需要在 Forward 表增加 `forward_type` 字段

2. **多协议选择功能（TCP + UDP 同时支持）**
   - 当前 Forward 模型缺少协议字段
   - 需要增加 `protocols` 字段支持多协议

3. **按用户/隧道的流量配额控制**
   - 已有 UserQuota 表，但需要完善相关功能

4. **限速功能**
   - 已有 SpeedLimit 表，需要完善限速逻辑

5. **单向/双向流量计费配置**
   - 需要增加计费配置

6. **链式隧道功能**
   - 已有 ChainTunnel 表，需要完善前端 UI

7. **UI 交互优化**
   - 需要优化页面布局和交互体验

8. **监控信息增强**
   - 需要增加实时监控和图表展示
