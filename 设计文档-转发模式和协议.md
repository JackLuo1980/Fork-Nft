# 转发模式和多协议设计文档

> 创建时间: 2026-03-21
> 目标: 实现端口转发与隧道转发模式切换，支持多协议（TCP/UDP）选择

## 一、需求分析

### 1.1 转发模式

根据 flux-panel 的功能，转发应该支持两种模式：

#### 端口转发（Port Forward）
- 用户指定入口端口和目标地址
- 直接转发流量到目标地址
- 适合简单的端口映射场景
- 配置示例：
  ```
  入口: 8080
  目标: example.com:80
  ```

#### 隧道转发（Tunnel Forward）
- 通过隧道链进行转发
- 支持链式隧道（多跳）
- 更安全，支持加密
- 配置示例：
  ```
  入口: 8080
  隧道: MyTunnel
  目标: example.com:80
  ```

### 1.2 多协议支持

支持以下协议组合：
- **仅 TCP**
- **仅 UDP**
- **TCP + UDP**（同时监听两种协议）

type Forward struct {

## 四、节点下发逻辑

### 4.1 端口转发模式

```json
{
  "forwarder": {
    "id": 1,
    "name": "My Forward",
    "type": "port_forward",
    "protocols": ["tcp"],
    "ports": [
      {
        "port": 8080,
        "protocol": "tcp",
        "target": "example.com:80"
      }
    ],
    "engine": "gost"
  }
}
```

### 4.2 隧道转发模式

```json
{
  "forwarder": {
    "id": 1,
    "name": "My Forward",
    "type": "tunnel_forward",
    "protocols": ["tcp"],
    "tunnel": {
      "id": 1,
      "name": "My Tunnel",
      "type": "tls",
      "chain": [
        {
          "nodeId": 1,
          "port": 443
        },
        {
          "nodeId": 2,
          "port": 443
        }
      ]
    },
    "ports": [
      {
        "port": 8080,
        "protocol": "tcp",
        "target": "example.com:80"
      }
    ],
    "engine": "gost"
  }
}
```

### 4.3 多协议处理

当 protocols = "both" 时，需要创建两个转发规则：
- 一个 TCP 规则
- 一个 UDP 规则

### 4.4 引擎适配

不同引擎需要适配不同的转发模式：

#### GOST 引擎
- 端口转发：使用标准的 forward 转发器
- 隧道转发：使用 chain + forward 组合
- 多协议：创建多个监听器

#### NFTABLES 引擎
- 端口转发：使用 DNAT + SNAT 规则
- 隧道转发：暂不支持（或需要特殊处理）
- 多协议：为 tcp 和 udp 分别创建规则

#### REALM 引擎
- 端口转发：使用标准配置
- 隧道转发：暂不支持（或需要特殊处理）
- 多协议：创建多个监听端点

### 5.2 转发列表展示

在转发列表中增加显示：
- 转发模式标签（端口转发/隧道转发）
- 协议标签（TCP/UDP/TCP+UDP）

```tsx
<Table>
  <Column title="名称" dataIndex="name" />
  <Column
    title="模式"
    render={(_, record) => (
      <Tag color={record.forwardType === 'port_forward' ? 'blue' : 'green'}>
        {record.forwardType === 'port_forward' ? '端口转发' : '隧道转发'}
      </Tag>
    )}
  />
  <Column
    title="协议"
    render={(_, record) => {
      const protocolMap = {
        'tcp': 'TCP',
        'udp': 'UDP',
        'both': 'TCP+UDP'
      };
      return (
        <Tag color="orange">
          {protocolMap[record.protocols] || record.protocols}
        </Tag>
      );
    }}
  />
  <Column title="隧道" dataIndex="tunnelName" />
  <Column title="目标地址" dataIndex="remoteAddr" />
  <Column title="端口" render={(_, record) => record.ports.map(p => `${p.port}(${p.protocol})`).join(', ')} />
  <Column title="引擎" dataIndex="engine" />
  <Column title="状态" render={(_, record) => (
    <Tag color={record.status === 1 ? 'success' : 'error'}>
      {record.status === 1 ? '运行中' : '已停止'}
    </Tag>
  )} />
  <Column title="操作" render={(_, record) => (
    <Space>
      <Button onClick={() => onEdit(record)}>编辑</Button>
      <Button onClick={() => onDelete(record)}>删除</Button>
    </Space>
  )} />
</Table>
```
